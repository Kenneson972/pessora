2B — Gating des avantages Óra+ (prix réduits boissons)

oraMemberUnitPrice() retourne actuellement toujours le prix réduit (-50%). Il faut la conditionner à un abonnement Óra+ actif.
Approche A — Hook useIsOraPlus()
Hook pur qui retourne un boolean
subscription?.plan === 'ora_plus' && subscription?.status === 'active'
Utilisé dans les composants qui affichent le prix
✓ Simple, lisible, testable
✗ Chaque composant doit l'importer
Approche B — Recommandée ★
Même hook useIsOraPlus() — mais aussi expose
effectiveUnitPrice(pub: number) dans le hook
→ retourne prix réduit si Óra+, sinon prix public
Les composants n'appellent plus oraMemberUnitPrice() directement
✓ Logique centralisée, pas de duplication
✓ Facile à changer la règle plus tard
Résultat visible : si l'utilisateur n'est pas Óra+ actif, le prix boisson affiché dans le panier reste le prix public (pas de réduction). Le calcul s'effectue côté client uniquement pour l'affichage — le vrai prix est validé côté edge function.
